兩百字內的個人簡介

王鈺婷申請履歷 活潑外向、對未知的事物抱持好奇的態度，讓我總對生活週遭的事務充滿興趣。我從中興大學景觀與遊憩畢業一年多，現於桃園市政府建築管理處就職，但因不滿足於工作現況的安逸性，我於今年12月31日辭去此份工作，即便打破現狀，意味著兩日後，我便失業啦！！！ 想轉職成為工程師，大概源於對撰寫程式的嚮往，而實際學習後，寫出真正跑得動的程式帶給我的成就感，也讓我更加明確的決定了轉職這條路。 #為什麼想成為工程師 ( 請以自我期許帶入：前端、後端、或全端工程師 ) 現今的我想成為後端工程師，但其實最一開始我想從成為前端工程師開始。 作為使用者，網站、應用程式賞心悅目時，會大為提升我選擇該項目的意願，且前端的工作可以立即性的得到實質回饋，可以看見自己實際做了哪些設計，我認為快速的反饋對學習任何領域，成就感都將支持我更長遠的走下去；決定改變學習的面向，以成為後端工程師為目標，是源自於我突然理解到前端的工作如同一個人的外表，而後端的工作則如同一個人的內涵，在美麗的外表，倘若沒有內涵支持，那他也只能是花瓶，故開發專案時，最重要的應該是其後端內容，再者才是前端的介面優化等等。 #為了成為工程師，曾經做過什麼努力？ 大學時期，我曾參與為期一日的入門課程，但或許是對當時的我而言，此領域太過於陌生，就像鴨子聽雷，跟著講師做出了成品，卻無法應用於其他方面。 現在我從網路課程學習程式，從 Coursera、YouTube 上找影片學習，看不懂就再找其他講師的教學課程，再不懂就爬ppt，買書學習。  #對於工程師的職業想像是什麼？ 我理想中的工程師身著帥氣的西裝，隨身攜帶的公事包內裝著15寸的蘋果電腦，即便在假日都要隨時 On Call ，只要公司的網站突然發現 bug 就需要立即翻找程式碼，排除系統問題。但，以上感覺都只會是「理想」，希望來日我真的可以成為衣著光鮮亮麗的工程師。 #如果參與這個計畫，會怎麼安排學習時間？ 課程自明年2月中開始，而我再兩天便正式離開現職。在正式課程開始前，我會先觀看講師頻道內的影片，和觀看之前買來還沒看完的 Python 書籍；待課程正式開始後，每日準時咖啡廳報到，開線上會及上課程，全職專心準備。 #如果參與這個計畫，預期會碰到什麼困難？你打算怎麼解決它？ 我想參加這個計畫，我在課程中應該仍然會有很多聽不懂的地方，就像先前我自己觀看線上課程時會面對的問題，但我想我可以揪其他學員辦讀書會，彼此分享自己解決問題的方式、提出自己遇到的問題。 #想要對我們說的事情？ 對於培育新入門的人才總是一件很艱辛的工作，你們願意做這件事，真的很讓人敬佩！！
